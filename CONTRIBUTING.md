# Contributing to arx-core

arx-core is a generic telemetry-pipeline library extracted from
ArxSentinel. Contributions that keep the module generic and
self-contained are welcome — anything that introduces a product-specific
assumption (security scoring, vendor integrations, deploy tooling) does
not belong in this subtree and should go into the product layer instead.

## Adding a package

- Place the new package under `arx-core/pkg/<name>/`.
- Every package MUST ship a `README.md` containing: a one-paragraph
  purpose statement, the signatures of all exported types and functions,
  and one usage example.
- A package MUST NOT import anything from the arxsentinel product layer
  (`pkg/`, `internal/` outside `arx-core/`). The boundary is enforced by
  ADR-002 and verified by `go list -deps`.
- All text and Go comments inside `arx-core/` MUST be English. No
  Cyrillic characters in any `.md` file or Go source file under
  `arx-core/`.

## Genericization rule

arx-core is a framework, not a product. Code, comments, and
documentation inside `arx-core/` MUST stay security-product-agnostic.
In particular, do not introduce arxsentinel-specific examples such as
detector-threat scoring, IP whitelists, Cloudflare / MikroTik / sentinel
plumbing, or scorer logic into the core docs or tests. Product-specific
examples live in the arxsentinel product-layer documentation; the core
docs use neutral language (events, lines, plugins, registries, queues).

## Plugin development

Plugins — source, detector, sink, and executor — are the primary
extension point of arx-core. The interface contracts, registration
mechanism, blank-import pattern, manifest format, and lifecycle hooks
are documented in [`docs/plugin-development.md`](docs/plugin-development.md).
That file is being populated in a later phase of Flow 082; in the
meantime, the source of truth for each contract type is the corresponding
`Registry` package in `arx-core/pkg/` (source, detector, sink, executor,
processor).

## Build and test

Before opening a change:

```bash
go build ./...
go test -race -count=1 ./...
```

Per-PR grep gates:

```bash
# English-only invariant: must be empty.
# The class below is the Cyrillic block (U+0400..U+04FF) expressed in
# PCRE hex notation, so the gate itself stays free of Cyrillic bytes.
grep -rlP '[\x{0400}-\x{04FF}]' arx-core/ --include='*.md'

# arxsentinel references: only intentional mentions (CONTRIBUTING/README
# disclaimers), never import-shaped mentions
grep -rli arxsentinel arx-core/ --include='*.md'
```

## Commit style

Follow conventional commits (`feat:`, `fix:`, `docs:`, `refactor:`,
`chore:`, …) with an imperative subject line. Always pass
`--no-co-author` to `git commit`. Do not add AI attribution trailers
(`Co-authored-by: claude`, `Generated by`, etc.) — the project
identity is the repository owner only.
